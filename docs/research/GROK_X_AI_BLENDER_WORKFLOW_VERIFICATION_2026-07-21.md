# Grok / X AI+Blender Workflow Verification and Application

**Date:** 2026-07-21
**Scope:** Every creator/method materially named in the supplied Grok share plus the user's direct `@posi_posi8`, `@hos_giken`, `@poistudioltd`, and `@MrLarus` sources.
**Method:** direct X posts/media, primary repositories/papers/product docs, adversarial source grading, no engagement-as-quality inference.

## Executive result

The Grok synthesis was directionally useful but over-broad. The evidence supports five transferable methods:

1. high-level brief plus small, checkpointed agent operations;
2. reference packs and semantic component generation;
3. Blender/DCC as authoritative assembly, topology, rig, lookdev and validation layer;
4. separate agent/model lanes for modeling, camera/previs, critic and making-of;
5. AI output as candidate/reference/teacher data, never automatic game/runtime/truth authority.

No direct source supports one-prompt production-ready characters, deterministic combat motion, or replacing runtime MotionBricks/physics truth.

## MrLarus / Depth Anything / Seedance / Grok combat proposal

Direct MrLarus posts claim depth-video conditioning preserves visible dance/martial-arts motion and can reduce moderation false flags before generated-video character/scene replacement. They do not claim skeletons, metric depth, contacts, collision, physics or deterministic replay.

Primary-source result:

- Depth Anything V2 is monocular image depth; video examples are framewise. Relative depth is not a metric skeleton.
- Seedance 2.0 is generated video, not motion truth.
- ARDY is a streaming interactive-motion research candidate but does not establish Just Dodge two-fighter/weapon/contact/determinism parity.
- MotionBricks remains the sole live presentation authority; deterministic physics remains truth.

Applied:

- added `combat-motion-teacher-corpus/references/depth-video-seedance-boundary-2026-07-21.md`;
- depth is an offline auxiliary teacher channel only;
- required held-out RGB vs depth vs RGB+depth vs contact-aware-retarget ablation;
- Seedance limited to labeled synthetic visual concept/reference augmentation;
- no recovered/generated clip becomes runtime fallback or combat truth.

## Canonical operational loop

```text
brief + licensed references
→ semantic component candidates
→ typed Blender/DCC micro-operations with checkpoints
→ topology/articulation/weights/PBR/material/collision QA
→ save + GLB round trip + diagnostic renders
→ target runtime proof
→ human visual/promotion decision
```

The toolchain proved this control shape on Blender 5.2.0 LTS / DCC 0.19.62 with a typed sword smoke asset, GLB round trip, material/UV/mesh/export gates and corrected render. That artifact is toolchain evidence only; it is not a Just Dodge production asset.
